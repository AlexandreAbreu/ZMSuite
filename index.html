ZMSuite by AlexandreAbreu

Implementations of a suite of tools used to deal w/ Infocom games (zmachine implementation, ZIL interpreter/compiler, etc.)

Infocom was the game company behind well known adventure games like Zork I-II and III. Those games featured a text based interface in which you would type commands and have a response describing the result of that command being printed. The Infocom game development environment is very rich and interesting for many reasons: - all games are 'story files' that are basically snapshots of the initial status of the whole game environment (memory), - the code for each game is NOT native code but an Intermediate Language that is then being run at runtime against a Virtual Machine: the zmachine, - Infocom created a specific Lisp inspired languages to create story files and stories in general. This language fist the domain and helps properly create the objects and the interactions between those world objects. The whole design is pretty clever, not too complex and fun to work with. It is a great way to work on something fun AND be involved in VM implementation (light one) and potentially more (lisp interpreter, IL disassembly etc.).
